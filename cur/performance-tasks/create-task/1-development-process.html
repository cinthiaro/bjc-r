Usando un proceso de desarrollo para organizar tu código

En la práctica Tarea de creación, te familiarizarás con las guías y el vocabulario de la AP Tarea de creación, y practicarás la realización de la tarea con la guía y la retroalimentación de tu profesor.

En esta página, reflexionarás sobre tus hábitos de programación y planearás el proceso de desarrollo que usarás para la Tarea de creación.

Tarea de creación es para que presentes algo al consejo universitario, para incluirlo en tu puntuación del AP CSP que sea creativo: un programa que diseñes (con o sin un compañero), un video del programa ejecutándose (desarrollado por ti mismo), y una respuesta escrita sobre tu programa y tu proceso de desarrollo (escrito por ti mismo).

CRD-2.E.1, CRD-2.E.2

El proceso que utilices para desarrollar tu programa puede ser organizado y estratégico o puede ser un proceso de exploración. Hay diferentes maneras de programar. Aquí hay algunas fases comunes por las que pasa un programador:

    CRD-2.F.1
  • investigando y reflexionando—averiguando los requisitos: lo que el usuario quiere que haga el programa
  • diseñando—la planificación de cómo será la aplicación para el usuario y el enfoque general que se utilizará para programarla
  • prototipos—construyendo un borrador del programa
  • Probándolo—probarlo, compartirlo con otros para obtener retroalimentación, arreglar cualquier error, y hacer cualquier mejora necesaria

CRD-2.E.3, CRD-2.E.4

Tu proceso de desarrollo podría ser iterativo (que implica varios ciclos de desarrollo y luego pruebas y retroalimentación) y/o puede sr incremental (lo que significa que se construyen y prueban piezas más pequeñas del proyecto antes de combinarlas).

  1. Piensa en cómo programas ¿Sueles programar de forma iterativa (programando y luego probando y luego volviendo a programar, etc.)? ¿Sueles programar de forma incremental (construyendo y perfeccionando una pieza a la vez? ¿Hay algún problema de desarrollo con el que se hayas encontrado una y otra vez?  ¿Hay alguna forma en que pueda cambiar tu proceso de desarrollo para evitar esos desafíos?

    2. CRD-2.E: Desarrollar un programa usando un proceso de desarrollo.
  2. Antes de comenzar tu Tarea de creación, decide qué tipo de proceso de desarrollo usarás. Escribe tu plan. Está bien si cambias el plan a medida que avanzas, pero mantén un registro de tus cambios también para que puedas escribir sobre ellos para la respuesta escrita que tendrás que presentar.

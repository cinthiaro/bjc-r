Aquí hay tres posibles casos a considerar, no sólo el caso base habitual o el caso recursivo. Todavía hay un caso base, es decir, una lista de entrada vacía, en cuyo caso reportaremos una lista vacía. Pero hay dos casos recursivos, dependiendo de si el primer elemento de la lista es o no par. (Tiene que haber un primer elemento, si no estamos en el caso base.) Los bloques verdes de Operadores de arriba, toman el restante al dividir el primer número por 2, y ven si ese restante es 0, en cuyo caso el número es par.

Si el primer número es par, entonces queremos incluir ese número en el resultado. Así que reportamos ese número en frente de la llamada recursiva en el resto de los números.  Si el primer número no es par, entonces no queremos incluirlo en el resultado, así que simplemente reportamos el valor reportado por la llamada recursiva.

Como en los plurales, el código de los pares es un ejemplo de un patrón de código. No hay nada especial en los números pares aquí. El mismo patrón podría usarse para hacer una lista de números impares, o números que terminan en 7, o nombres que empiezan con una Z.

Realiza los siguientes ejercicios utilizando recursión, no las funciones de orden superior.

    Intenta modificar el código de los pares.
  1. Escribe un bloque ends-e que tome una lista de palabras como entrada, y reporte una lista de esas palabras de la entrada cuya última letra es e.
  2. Escribe un bloque de números que tome una lista de palabras y números mezclados como entrada, ay reporte una lista de sólo los números de la lista de entrada.
  3. Generaliza el patrón para hacer un bloque de mantener.
    4. Si no trabajaste el bloque combinar en la página Unidad 5 Laboratorio 3 Página 1: Comparando algoritmos, probablemente querrás tener una idea de cómo funciona ese bloque antes de empezar el siguiente problema.
  4. Escribe el bloque combinar. Nota: El caso base será una lista con un elemento, no una lista vacía.
